# MediaPlayer 交互规范 V1.5

Last updated: 2026-03-01

## 主体布局

- 根布局固定为 `Header + Sidebar + Main`。
- Header、Sidebar、Main 的比例均可由用户调整。
- Sidebar 与 Main 之间提供可拖动分割条，用于实时调整布局比例。
- Sidebar 宽度低于 `3%` 时自动折叠，仅显示单个“展开目录”按钮。
- 折叠态“展开目录”按钮位于 Sidebar 容器垂直居中，箭头朝右，颜色为灰色，透明度从左到右递减；悬停显示文字提示。
- 应用启动时优先渲染主界面框架占位（Header / Sidebar / Main / Metadata）；后端数据异步加载，不阻断窗口显示。
- Main 区包含：
  - 分页内容区
  - 可折叠元数据面板
  - 当前焦点 (focus) 项信息 Footer

## 当前阶段实现说明（实链路）

- 当前阶段以真实后端数据驱动交互为主，虚拟数据仅保留测试回退用途。
- Main/Metadata/Fullscreen 默认走真实图片/视频渲染链路；资源不可用时回退占位态。
- 视频时长/分辨率已接入真实探测，播放器在 `loadedmetadata` 事件回传后执行进度校准。
- 图包评分与保存封面 (Save as cover) 已接入后端写链路，前端采用乐观更新 (optimistic update) + 失败回滚。
- 导入动作已接入任务队列；任务状态在页面任务面板中可见，失败项支持重试。
- 运行时依赖不足时，页面顶部显示“降级策略已生效”告警，并列出受影响能力与回退行为。
- 验收目标是“布局、控件位置、快捷键作用域、状态联动”与本文档一致。

## Header 控件

- Logo 作为统一导入入口。
- Logo 菜单提供“导入文件”“导入文件夹”两类弹窗入口。
- Header 在 Logo 与模式切换之间显示“任务状态按钮”（`加载中/空闲`）；点击后切换导入任务面板显示状态。
- 混合输入通过“全窗口拖拽”与“粘贴路径”实现。
- 检测到文件/目录拖拽进入窗口时，显示全窗口导入叠加层；释放后自动入队并刷新任务状态。
- 模式切换：图片模式 / 视频模式 / 音乐模式。
- 检索入口：Header 仅保留“检索”按钮；可用性由当前模式与交互锁状态共同控制。
- Header 新增“管理”按钮：点击后展开管理面板；管理面板与检索面板互斥，进入管理模式会自动退出检索模式。
- 管理面板采用精简布局：不显示额外模式头行与操作说明文案，仅保留选择统计、操作按钮与执行结果提示。
- 点击“检索”按钮展开检索面板。
- 检索面板支持折叠；折叠后不退出检索模式，以上方居中“下箭头”按钮恢复展开。
- 检索面板当前提供多字段筛选：`名称`、`作品名`、`社团`、`作者`、`标签(tags)`、`评分`。
- `名称`、`作品名` 支持模糊匹配。
- `社团`、`作者` 输入框基于库内值提供自动补完候选。
- `标签(tags)` 通过标签选择面板点选（支持多选）。
- `评分` 通过“`×(无评分) + 1~5 星`”筛选；选择第 N 星时，前置星同步高亮。
- 多字段组合采用 AND 逻辑过滤结果；单字段时仅按该字段输出结果。
- 检索面板与下方工作区之间提供可拖动分割条，用于调节检索面板高度。
- 调色板（日/夜）切换、设置、帮助、窗口控制（最小化/最大化/关闭）统一由 Header 提供。
- 缩放级别与自动播放控制不在 Header 直接提供，使用主区控件与快捷键链路。

## Sidebar 行为

### 图片模式

- 展示分层根目录、子目录与图包节点。
- 不展示图片文件叶子。
- 任意节点可设置为“当前根 (Current Root)”，用于范围化浏览。
- 目录结构栏右侧始终使用统一方形图标按钮切换“设为根/恢复根目录”；仅在“设为根”态且当前选中节点为目录时可用。
- 若已设置根目录，标题区直接显示根目录名称。
- 点击 Sidebar 标题按钮可切换树显示模式：`直属媒体父子结构` 与 `实际路径层级结构`。
- 目录节点右侧显示“当前目录直接包含的图片数”，不累计其子目录与压缩包内目录。
- 压缩包中的目录按同样规则展示直接图片数。
- Sidebar 节点最左侧显示加载状态圆点：`pending` 显示圆点，`running` 显示高亮圆点并高亮行背景，加载完成不显示圆点。
- 当目录仅包含图片时，Main 直接显示该目录对应图片缩略图。
- 主界面分割条（Sidebar/Workspace、检索面板/主区、Main/Metadata）受“布局锁定”开关控制：锁定时不可拖动。
- 检索结果视图下，Sidebar 内容保持“路径/节点树”形态，不重复列出同一节点；仅最末级命中叶节点显示检索结果数量，上级目录不累计显示。
- 检索结果视图下，Sidebar 与主区焦点同步遵循“由当前结果上下文驱动”的规则；不允许出现节点高亮与主区焦点脱节。
- 检索结果视图下，若 Sidebar 可视区外的节点被同步高亮，Sidebar 需自动滚动到可见位置。
- 检索结果视图下，Sidebar 头部将“设为根”替换为“转到”；点击“转到”后退出检索模式，恢复常规 Sidebar 排序，并定位到当前焦点所在节点。
- 图片模式常规浏览下，Sidebar 头部提供“一键折叠含图父级节点”按钮，用于折叠所有实际包含含图目录/压缩包的父级节点。
- 图片模式常规浏览下，Sidebar 头部提供“上一个/下一个含图父级节点”按钮，用于在含图父级节点间顺序跳转；到达边界后对应按钮禁用，不循环。
- Sidebar 列表区保持“无可见滚动条但可滚动”的交互（滚轮/触控板/键盘滚动有效）。
- 管理模式下，Sidebar 每个节点左侧显示勾选框 (checker，左对齐)，支持单选与 `Shift + 点击` 区间选择。
- 管理模式下，勾选父节点会自动勾选全部子孙节点；取消父节点勾选会自动取消其全部子孙节点勾选。
- 管理模式下可执行 Sidebar 选中节点删除：触发危险确认弹窗，需勾选“我了解此操作将永久不可逆地删除选中数据”后方可确认。

### 视频模式

- 展示分层根目录、子目录与视频文件叶子。
- Sidebar 可折叠目录节点右侧显示“直属视频子节点数”（样式与图片模式计数徽标一致）。
- 点击可折叠目录节点进入 Main 的“节点浏览缩略图容器”，展示该目录直属视频子节点封面。
- 在该浏览态点击缩略图或点击 Sidebar 子节点，都会切回 `video-preview + video-controls-shell` 并立即开始播放所选视频。
- 视频节点浏览缩略图使用与图片模式一致的自动分页语义与 `1:1` 卡片约束。
- 视频节点浏览缩略图支持与图片模式一致的缩放（scale）Popover 控制。
- 视频节点浏览缩略图交互：`单击/方向键` 仅切换元数据焦点；`双击/Enter/Space` 才切回播放态并开始播放。
- 每个视频叶子带 `toggle`，用于加入/移出全屏播放列表。
- 播放列表为全库共用。
- 管理模式在视频模式同样可用：支持 Sidebar 节点勾选框与目录/文件级删除。
- 管理模式工具栏提供“视频转码 (TC)”入口：可对当前管理勾选视频（含 Sidebar 节点解析出的批量视频）或焦点视频发起任务化转码。
- 视频转码面板支持容器/编码器/质量模式/分辨率与帧率/音频策略/输出目录参数，打开面板会读取能力矩阵并对不可用选项做灰度禁用。
- 视频转码面板提供体积预估（参数变更后 debounce 刷新），展示源大小、预计输出、估计范围与压缩比。
- 视频转码任务执行中支持取消；任务结束后可直接“打开输出目录”。

### 音乐模式

- 展示分层根目录、子目录与音频文件叶子。
- 音频叶子可参与 Sidebar 勾选框选择（管理模式），并复用目录/文件级删除链路。
- `series_id` 匹配成功时，音乐主工具栏显示“漫画版/动画版”跳转按钮。

## Main 行为

### 图片模式

- 使用分页缩略图网格，不使用无限滚动；包含 folder 父级节点的节点浏览视图同样遵循分页语义。
- 翻页控件统一迁移到 Main Footer 右侧（右对齐），主内容区不再单独渲染 `pager-line` 容器。
- 支持“缩略图模式”与“纯文件名模式”切换。
- 缩略图模式默认卡片比例固定为 `1:1`，分页内按行列顺序排布（不改变每行从左到右顺序）。
- Main Toolbar 右侧操作按钮改为固定方形图标按钮（无文字直显），通过图标状态表达模式。
- Sidebar Head / Main Toolbar / Metadata Head 标题字号与头部高度保持一致，三栏标题横向对齐。
- 缩略图模式优先请求 `thumbnail` 变体资源；元数据预览与全屏继续请求 `original` 变体。
- 媒体解析请求采用限并发队列，焦点原图（元数据/全屏）优先于当前页缩略图请求，避免高速翻页时原图长期饥饿。
- 对焦点附近图片启用近邻原图预取（高优先级、受并发上限约束），用于提升连续翻页命中率。
- 翻页/切包/筛选触发加载时：若当前无可用快照则显示 loading skeleton；若已有上一页快照则保留上一页内容并等待新页 commit。
- 翻页加载中若已有上一页快照，Main 保留上一页缩略图显示并仅更新加载状态提示，避免连续翻页时网格闪空。
- 缩略图内图片按长边适配并居中：`landscape / square` 垂直居中，较窄 `portrait` 水平居中。
- 缩略图卡片内不显示 caption 文本，避免不同标题长度影响网格布局。
- 缩放算法采用离散级别：仅 7 级，初始为第 4 级；初始级别以“缩略图宽度设置值的 75%”作为锚点自适应选取。
- 缩放 Popover 数值与网格纵向行数保持直映射：`level = rows`（`1→1 行`，`7→7 行`）。
- 纵向滑条方向统一由轴容器旋转控制（axis rotate）；不在 runway 本体叠加 reverse 反转。
- 缩略图网格采用“纵向优先贴合”布局：优先保证网格高度贴合容器，避免底部空白/裁切。
- 拖动主界面水平分割条过程中不触发缩略图吸附；仅在 `mouseup` 后执行一次横向吸附。
- 调整缩放级别后会执行一次横向吸附；吸附过程应一次到位，不允许可见的来回弹跳。
- 节点浏览（非图包节点）缩略图与图包卡片统一为纯封面形态，不显示名称与数量文案，避免因卡片高度差异触发额外布局重算。
- 纯文件名模式改为文件列表视图，不分页，改为滚动浏览。
- 纯文件名模式展示真实文件大小与分辨率；若无法探测分辨率则显示 `-`。
- 管理模式下，缩略图与纯文件名行均显示勾选框；支持点击点选与鼠标框选。
- 管理模式下，缩略图模式无需精确点击勾选框，点击缩略图卡片即可触发该项勾选/取消勾选。
- 管理模式下，缩略图模式支持“按住左键拖动批量反转”交互：鼠标经过的缩略图在同一次按压期间仅反转一次（松开后重新开始下一轮）。
- 缩略图/文件名选择状态以图片 id 持久记录，翻页、切换文件夹或压缩包后仍保留。
- Sidebar 勾选框与缩略图/文件名勾选框互斥：任一侧产生新选择时，另一侧选择会自动清空。
- 管理模式提供“隐藏/取消隐藏”（仅作用于缩略图/文件名选择项）；隐藏项在非管理模式不显示，且不计入 Sidebar 图片计数。
- 管理模式执行图片删除时，目录图片按文件级删除；压缩包图片按“保留项重打包 -> 校验 -> 替换源包”流程删除，并同步清理数据库映射与缩略图缓存。
- 管理模式重命名支持“先预览后执行”：预览结果需显示源名、目标名与失败原因（如有），并与实际执行结果保持一致。
- 管理模式 `R` 在存在管理选区时进入批量重命名面板：可对 Sidebar 勾选节点与图片勾选项执行统一重命名。
- 批量重命名模式包含：`replace`、`numbering`、`remove-range`、`metadata`；`metadata` 模板可编辑。
- `remove-range` 交互采用“从第几位到第几位删除字符”（1-based，含端点）。
- 管理删除执行后，管理面板需显示结果提示；若存在部分失败，提示文案需显式包含“已删除 N（项/张），失败 M 项”。
- 管理模式新增“广告审核”动作：以当前管理选择范围作为输入，触发 LLM 广告审核任务。
- 审核完成后展示“疑似广告图片”复核列表，支持逐项勾选/取消、全选候选、清空候选与关闭结果。
- 广告审核结果在用户确认前不会触发物理删除；删除按钮会先进入危险确认弹窗，确认后复用既有 `deleteImageItems` 链路执行。
- 已确认删除候选会写入“图片哈希缓存”（known-hash）；后续命中可直接标记疑似并跳过 LLM。
- 设置面板“模型参数”分组提供广告审核策略参数：`all/head-tail`、并发度，以及 head/tail/streak 窗口参数（head-tail 模式显示）。
- 管理面板的广告审核卡片显示审计统计：来源分布（known-hash / llm suspected-clean-failed / strategy-skip）与命中率（LLM/总体）。
- 支持点击与键盘移动焦点。
- 元数据面板随焦点实时更新。
- 检索模式下 `metadata-search-section` 固定显示在元数据编辑区顶部（位于 `metadata-editor-shell` 顶部），不再位于底部。
- 元数据面板在展开态由标题“元数据面板”本身作为折叠入口；折叠态使用侧边箭头按钮恢复展开。
- 当前焦点图片 caption（图包名+序号+文件名）显示在元数据预览区底部。
- caption 中的文件名支持自动换行与自适应高度，默认完整显示，不做省略截断。
- 元数据原图切换采用双缓冲：新图完成解码前保留上一帧，避免翻页时黑屏/坍缩跳闪；切换期不额外叠加“加载中”文案。
- 元数据头部增加“原图/元数据显示”切换图标按钮（右对齐方形按钮）。
- 元数据显示态首项为评分星级组（复用 `feature-rating-group`）：采用“`×(无评分) + 1~5 星`”；默认无评分（不点亮星星）；支持点击设置与按住左键拖动连续选星。
- 评分行中点击非星星区域会清空评分（回到“未评分”状态）。
- 元数据显示态支持编辑并写入数据库：`作品名`、`社团`、`作者`、`标签(tags)`；`图包名`只读。
- 元数据显示态增加“动画版”按钮：当图片元数据 `系列ID(series_id)` 可匹配到视频元数据时可点击，点击后切换到视频模式并定位目标视频。
- 元数据显示态提供“作品名同步图包名”按钮：同步后仅替换主名，保留原后缀（如 `.zip`）。
- 元数据管理态增加 `系列ID` 输入框，支持按 Enter 写入并持久化。
- 普通元数据显示区新增“行为分析记录（Preference Metrics）”折叠面板（默认折叠）：`行为次数`、`已读页数`、`完成度`、`最近行为时间`；该区仅用于观察，不提供编辑入口。
- 图片类行为事件仅在“图片模式 + 全屏”会话中开始采集；仅点击节点加载缩略图不计入行为。
- 图片类行为指标仅在会话结束时落库：触发条件为“退出全屏”“切换到其他模式全屏”“退出 App”。

### 视频模式

- 提供视频预览区域与播放控制。
- 当选中可折叠目录节点时，Main 内容区可切换为节点浏览缩略图容器（复用图片模式缩略图卡片容器与样式）。
- 若视频尚未保存封面，系统会自动以首帧生成并保存为封面。
- 包含“保存封面 (Save as cover)”按钮，用于手动保存封面。
- “保存封面 (Save as cover)”采用乐观预览更新，最终结果以后端回写为准。
- 视频信息视图支持写入数据库：`评分`、`作品名(EN/JP)`、`社团(EN/JP)`、`作者(EN/JP)`、`标签(tags)`。
- 视频信息视图增加“漫画版”按钮：当视频元数据 `系列ID(series_id)` 可匹配到图片元数据时可点击，点击后切换到图片模式并定位目标图包。
- 视频信息视图支持“同步文件名到作品名”，会用去扩展名文件名覆盖当前 `作品名`。
- 视频元数据默认只读态中，`作品名/社团/作者` 采用本地化展示（EN/JP 切换）且值可点击触发静默检索。
- 视频元数据编辑区保留 `系列ID` 输入框，支持按 Enter 写入并持久化；默认只读态不展示 `系列ID` 字段。
- 普通视频元数据显示区新增“行为分析记录（Preference Metrics）”折叠面板（默认折叠）：`行为次数`、`观看时长`、`完成度`、`最近行为时间`；该区仅用于观察，不提供编辑入口。
- 视频行为事件在全屏/非全屏均可采集；非全屏会话 `<10s` 记为噪音并写入会话事实层，标记 `is_noise=1`。
- 视频行为指标采用“延迟写入”策略：播放中仅在内存累计，`停止播放`、`切换节点`、`切换模式`、`退出 App` 时一次性落库。
- 系统设置页新增“偏好调试面板”：用于查看 `xp_preference_metrics_v1` 最近一次写入原因、聚合条目计数与最近会话事件样本（图片/视频各 8 条）。
- 系统设置页「调试设置」新增 `Debug` 开关：用于控制 Header 是否显示调试组（位于 g2 与 g3 之间）。
- Header 调试组包含三个按钮：`TT`（切换 Tooltips 显示）、`N`（切换 Electron 外框与菜单）与 `T`（打开界面参数面板；若当前为关闭态会先启用“显示界面参数按钮”）。三个按钮均提供 `on/off` 状态标识。

### 音乐模式

- Main 主区采用 names-only 列表（文件名/大小/时长），不提供缩略图网格。
- 播放控件固定为 `music-controls-shell`：音量、上一首、播放/暂停、下一首。
- 主区可视化区域改为 Shader 渲染（默认 `Default`，来源 Shadertoy `McsSzB` 单 pass），不再显示占位文案。
- 可视化支持两种渲染后端：`GPU(WebGL2 Shader)` 与 `CPU(Canvas2D fallback)`。
- 可视化运行时显示分辨率与 FPS 调试 HUD（由设置项开关控制）。
- 可视化区域支持全屏覆盖层；全屏态提供“退出全屏”按钮并复用 `Esc/F` 与右键关闭语义。
- 音乐元数据面板支持编辑并写入数据库：`album`、`author`、`trackTitle`、`seriesId`（Enter 提交）。
- 音乐元数据默认只读态仅显示 `album/author/trackTitle`，并支持点击触发静默检索。
- 音乐模式 `series_id` 匹配时可跳转到图片/视频模式，跳转后自动应用 `seriesId` 检索并定位首个匹配项。

## 导入任务队列

- 导入入口统一收敛到任务队列：原生文件弹窗、原生文件夹弹窗、全窗口拖拽、粘贴路径/文件。
- 任务状态最小集合：`pending`、`running`、`completed`、`failed`。
- 导入任务面板默认隐藏，仅在点击 Header 任务状态按钮后显示。
- 任务面板展示：来源、进度（processed/total）、状态、消息；失败任务提供重试按钮。
- 导入任务 `completed` 判定为“薄扫描 + SQLite 入库完成”；压缩包归一化等重操作继续在后台队列执行，不阻断主读链路。
- 导入进行中需实时更新任务消息：至少包含当前阶段与“已处理文件数”，用于调试与进度观测。
- 路径校验规则：空路径、不可读目录、非媒体类型文件会被标记失败并在错误详情中记录；导入为“纯引用”：库外路径不会复制入库，仅登记引用并在原路径扫描/读取媒体。
- `rar/7z` 归一化成功后会在源目录原地替换为 `.zip(store)` 并删除原包；未完成或失败时该包在 UI 侧保持占位/空结果，不阻断其它浏览。
- 任务完成后自动触发库刷新；成功与失败混合时判定为失败并保留详细失败原因。
- 切换 Sidebar 视频项后进入新视频封面暂停态，不继承旧进度。
- App 视频控制区采用两行结构：第一行为“当前时间/总时间 + 进度条”，第二行为三组按钮（左/中/右）。
- Popover（音量/字幕/速度/适应/播放列表）遵循单开契约：任意时刻只允许一个面板打开，鼠标移出“按钮+面板”wrapper 即关闭。
- 左组包含：音量（按钮+竖向滑条 Popover）、字幕（来源选择/状态提示 Popover）、速度（preset Popover）、全屏、适应（Popover）。
- 中组包含：上一个、播放/暂停、下一个。
- 右组包含：保存封面 (Save as cover)、切换独立/复合模式、播放列表。
- `切换独立/复合模式` 与 `播放列表` 在非全屏下禁用（保留 DOM 以稳定布局）；在全屏下可用。
- 倍速仍保留兼容滑条（`倍速滑条`）以兼容现有自动化断言。
- 元数据面板可切换：
  - 视频信息视图
  - 播放列表管理视图
- 视频模式下元数据面板布局与图片模式保持同层级结构；视频信息区包含 `评分区(可写, ×+1~5星)`、`文件名/作品名/社团/作者/标签(tags)` 与 `metadata-edit-actions(可写: 保存/同步文件名到作品名)`；`时长/分辨率/音量/倍速` 固定于底部信息区，与上方信息/列表区域分离。

### Main Footer

- 始终展示焦点项详细信息。
- 展示项包括路径、文件大小、分辨率。
- `zip` 图片路径格式：`压缩包绝对路径 + 序号`。
- 图片模式分页时（含节点浏览视图），Footer 右侧展示“上一页 / 页码 / 下一页”翻页控件。

## 播放列表交互

- 支持拖拽调整顺序。
- 支持直接删除条目。
- 在视频模式元数据播放列表中，条目获得焦点后可按 `Delete` 直接删除。
- 删除条目后，同步 Sidebar 对应 `toggle` 关闭。
- 若删除当前播放项，自动切换到下一可用项。

## 全屏模式

- Enter 或双击焦点项进入全屏。
- `F` (`KeyF`) 切换媒体全屏/窗口模式（图片模式与视频模式都可用）。
- 音乐模式同样支持 `F` 切换全屏；进入后显示“仅可视化”覆盖层，不启用图片/视频双显示层。
- `Shift+F` (`Shift+KeyF`) 切换 Electron 窗口全屏（仅在非媒体全屏态生效；媒体全屏态下忽略）。
- 从图片模式进入时默认 `仅图片`；从视频模式进入时默认 `仅视频`。
- 鼠标进入底部 20% 区域时显示“浮动叠加”Footer（仅图片全屏与双显示），不占用内容区高度。
- Footer 保留“双显示”切换与“调换左右”按钮：
  - 手动点击“双显示”后进入双显示。
  - 双显示态可切回单显示；回到单显示时始终恢复“进入全屏时的单显示模式”（图片进入则回图片，视频进入则回视频）。
- 双显示态移除固定分屏比例滑条，改为拖动中间分隔线实时调整比例。
- 双显示态下，焦点通过“点击图片/视频区域”或 `Tab` 切换；Footer 导航与对齐操作作用于当前焦点区域。
- 双显示态下，视频区域支持鼠标左键拖动；按视频比例自适应后，基于剩余留白空间可进行水平/垂直位置微调。
- 全屏图片/视频按实际宽高比自适应缩放。
- 全屏图片切换采用双缓冲：新图未就绪时保持上一帧，解码完成后再原子替换，避免“先黑后亮”；切换期不额外叠加“加载中”文案。
- 双显示态强制使用容器内自适应缩放，不继承单显示缩放值。
- 单显示态支持缩放与平移：
  - Footer 缩放按钮。
  - `Ctrl + 鼠标滚轮` 缩放。
  - 鼠标左键拖动平移。
  - `Reset` 恢复到自适应缩放。
- 全屏视频区域内，鼠标移入时显示悬浮播放控件，移出后隐藏；`仅视频` 模式不再显示 Footer，视频控制与播放列表入口统一放在该悬浮控件内，控件结构与视频预览态 `video-controls-shell` 对齐。
  - 控件仅两种锚点：贴视频底部或贴视频顶部（根据视频在容器中的垂直位置自动切换）。
  - 贴底部时顺序为“进度行 -> 播放控制行”（进度行贴近视频底边）。
  - 贴顶部时顺序为“播放控制行 -> 进度行”（进度行贴近视频顶边）。
- Footer 增加“上个包/下个包”按钮（图片链路）。
- 图包转换预览模式（`RS` -> 预览）复用全屏图片 Footer：
  - 预览态隐藏“双显示/调换左右/自动播放”，保留“上/下张、上/下包、对齐、缩放、退出全屏”。
  - 预览态新增 `S/F/Q` 参数按钮与 Popover（Scale/Format/Quality），修改仅作用于预览草稿。
  - 预览态新增 `AS`（Apply Scale）按钮：将“当前 Scale 作用后的最长边像素”写入 `Longest Edge` 草稿值。
  - 管理态 RS 面板新增 `Longest Edge(px)` 输入：留空时按 `Scale`；有值时按“最长边=目标像素”缩放并保持比例，且不放大小图（仅缩小）。
  - 当 `Longest Edge(px)` 有值时，全屏预览里的 `S` 控件禁用，避免与“最长边优先”策略冲突。
  - 预览态新增 `ADJ` 调节面板（大面板）：
    - `Basic`：Brightness + Contrast（仅显示并仅应用该模式参数）。
    - `Levels`：Input Black / Input White / Gamma（仅显示并仅应用该模式参数）。
    - `Curve`：Master 曲线编辑（线性对角线为默认无改动状态，节点支持 X/Y 调节；仅显示并仅应用该模式参数）。
    - 面板默认左下角显示，可拖拽移动位置，避免遮挡对比主画面。
    - 面板内提供 `Reset`（仅复位当前模式参数，不切换模式）与 `Cancel`（恢复本次打开 ADJ 的初始快照并关闭 ADJ）。
    - ADJ 打开时隐藏全屏 control-shell，且允许继续拖动对比分割条；面板外点击不关闭 ADJ。
    - ADJ 关闭入口仅 `Cancel` 按钮与 `Esc`（`Esc` 优先关闭 ADJ，而非退出全屏）。
  - 预览态 `OK` 后保存当前调节参数到会话，下一次调节以上次结果为基础继续。
  - 预览态中心区新增“确定/取消”；确定提交草稿参数到管理面板，取消丢弃草稿并退出预览。
  - 主显示区为“同图左右对比”：左侧原图、右侧转换预览图，中间分割条可拖拽。
  - 预览渲染失败时降级显示原图，并显示错误提示，不阻断继续浏览与退出。
- 图包转换执行态（`RS` -> 确定）进入后开启交互锁：
  - 管理工具栏除 `取消` 外其余入口禁用；参数控件与“预览/确定”禁用。
  - 全局快捷键与滚轮快捷键在执行态全部失效，仅 `Esc` 保留。
  - 执行中点击 `取消` 触发任务取消；未执行时点击 `取消` 仅退出 RS 模式。
  - 执行进度在 RS 面板内显示 `status + percent + message`。
- 退出全屏时，若自动播放处于开启状态，则自动关闭自动播放。

## 键盘控制

### 全局与图片浏览

- `Left/Right`：上一张 / 下一张。
- 非全屏 `Up/Down`：按网格上移 / 下移一个可视行步长；若越界则钳制到首项 / 末项。
- 全屏 `Up/Down`：跳到当前包首张 / 末张（不走网格步长）。
- `Ctrl+Left/Right`：上一个包 / 下一个包。
- 全屏图片态下，`Ctrl+Left/Right` 在 Footer 控件获得焦点时仍强制生效（不被输入控件吞掉）。
- 全屏下 `Alt+Up/Down/Left/Right`：对当前焦点区域执行上/下/左/右对齐。
- `A`：开始 / 停止自动播放。
- `1/2/3/4/5`：自动播放间隔 `1/2/3/5/8` 秒。
- `Alt+0..5` 或小键盘 `0..5`：图包评分（`0 => null`）。
- 非全屏下 `Tab`：在 Sidebar 与 Main 之间切换键盘焦点。
- 全屏且双显示下 `Tab`：在图片区域与视频区域之间切换 focus。
- 全屏图片翻页到边界时，继续按上一张/下一张会跨包进入上一个/下一个包；自动播放遵循同规则。
- 非全屏且焦点在 Sidebar 时，`方向键/Enter/PageUp/PageDown/Home/End` 用于 Sidebar 导航，不作用于缩略图翻页。
- `ArrowUp/ArrowDown` 移动到不可见项时，Sidebar 自动滚动保证目标项可见。
- `PageUp/PageDown` 以“当前可视项数量 - 1”作为步长翻页；超出边界时直接钳制到首项/末项。
- 鼠标点击 Sidebar 项后，焦点保持在 Sidebar；仅点击缩略图区或使用 `Tab` 才切回 Main 焦点。

### 视频控制

- 仅在“全屏视频”或“双显示且视频焦点激活”时生效。
- `Space`：播放 / 暂停。
- `PageUp/PageDown`：播放列表上一个 / 下一个视频。
- `Ctrl+PageUp/PageDown`：降低 / 提高播放倍速。
- `Home/End`：降低 / 提高音量。
- 视频转码任务进度更新采用节流写入（约 320ms），保障长任务期间 UI 轮询稳定。

## 设置面板形态

- 以约 80% 的居中面板形式打开。
- 背景内容轻度模糊但保持可见。
- 面板内部采用 `side | main` 分栏：左侧为设置分类，右侧为对应主内容区。
- 默认宽度配比为 `side 20% | main 80%`，横向占满设置面板。
- `main` 区沿用圆角面板外观并占满分配宽度；内部内容容器仅负责居中排版，不增加额外边框层。
- 左侧分类包括：界面设置、高级分页、AI模型设置、数据库设置、快捷键设置。
- AI模型设置新增“离线字幕模型档位”下拉：`current`（默认，保持现有 SenseVoice 链路）与 `funasr-nano`（调试档，走独立 FunASR 初始化与识别链路）。
- 当选择 `funasr-nano` 时，“一键下载模型”入口置灰并提示“当前档位仅支持本地扫描或手动指定目录”；`current` 保持既有下载行为不变。
- 设置面板不再包含独立 3D 分组。
- 布局参数页新增“布局锁定”开关：锁定后禁用主界面分割条拖动（不影响全屏模式拖动行为）。
- 布局参数页新增“容器边距系数”滑条（放在布局参数首行）：默认 `1.00`，范围 `0.00 ~ 3.00`，步进 `0.1`。
- 布局参数页新增“容器内边距系数”滑条（位于容器边距系数后）：默认 `1.00`，范围 `0.00 ~ 2.00`，步进 `0.1`。
- 布局参数页新增“容器内上中下间距系数”滑条（位于容器内边距系数后）：默认 `1.00`，范围 `0.00 ~ 2.00`，步进 `0.1`。
- 布局参数页移除“Header 高度系数”独立滑条，Header 高度改由按钮尺寸与容器节奏参数自动推导。
- 容器边距基准按“窗口宽度 `1%`”计算，并在运行时做像素级四舍五入（整数 px），统一驱动 Header/导入任务面板/Sidebar/Main/Metadata 五区边距与分割条宽度。
- 容器内边距基准按“窗口宽度 `1%`”计算，并在运行时做像素级四舍五入（整数 px），仅用于容器内部 padding。
- 容器内上中下间距基准按“容器内边距基准 `* 0.75`”再乘“容器内上中下间距系数”计算，并在运行时做像素级四舍五入（整数 px），用于统一 `toolbar ↔ main ↔ footer` 纵向节奏。
- 运行时变量：`--mpx-layout-gap-scale`（容器边距系数）、`--mpx-pane-inner-gap-scale`（容器内边距系数）、`--mpx-pane-stack-gap-scale`（容器内上中下间距系数）、`--mpx-layout-gap-px`（取整后边距 px）、`--mpx-pane-inner-padding-px`（取整后内边距 px）、`--mpx-pane-stack-gap-px`（取整后上中下间距 px）。
- 按钮与工具栏间距需同步消费容器相关变量（不可独立硬编码常量）；当按钮尺寸放大导致 head 区不足时，Header/Sidebar/Main/Metadata 的 toolbar 允许自适应增高（`min-height` 约束，不固定死高）。
- 布局参数页中的尺寸项改为“系数（x）”调节：界面显示为系数与换算后的像素值；窗口变小时主界面控件与文字自动等比缩放，降低布局破坏风险。
- 主界面窗口自适应缩放增加稳定阈值处理，避免在非最大化窗口下缩放值来回振荡导致字体/控件抖动。
- 各设置页主标题居中，输入框与滑条优先拉伸以完整显示参数内容。
- Sidebar 相关可调参数包括：最小宽度、主文字字号、数字字号、目录结构间隔。
- 目录结构间隔拆分为：多级目录水平缩进间隔与上下项间距。
- 界面设置新增“频谱可视化设置”分组：
  - `实际渲染长边分辨率`：控制 Shader 内部渲染尺寸（容器按比例缩放显示）。
  - `显示 FPS 调试信息`：显示 FPS、帧耗时、内部渲染分辨率与后端信息。
  - `渲染后端`：`GPU(WebGL2 Shader)` / `CPU(Canvas2D fallback)`。
- 高级分页新增“加载性能”分组（下拉配置）：
  - `相邻页预热范围`：`0/1/2/3`（`0` 为关闭相邻页缩略图预热）。
  - `预热批量级别`：`1/2/3/4`（用于控制每轮预热目标上限，不表示后端线程数）。
  - `全屏预取深度`：`2/4/6/8/12`（全屏焦点图附近原图预取半径）。
  - `全屏解码缓存数`：`4/6/8/10/16`（全屏已解码图缓存上限，采用 LRU）。
- 高级分页分组内为上述 4 个参数提供 Tooltip，统一用途为“解释参数作用范围、性能收益与资源成本”，用于降低误配概率：
  - `预热批量级别` Tooltip 必须显式提示“该值不是并发线程数”。
  - `全屏解码缓存数` Tooltip 必须显式提示“值越大内存占用越高”。
