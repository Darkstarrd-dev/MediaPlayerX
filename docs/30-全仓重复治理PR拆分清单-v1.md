# 全仓重复治理 PR 拆分清单 v1

Last updated: 2026-03-03

## 0. 使用说明

- 本清单用于按 PR 粒度推进“重复代码治理 + 回归风险控制”。
- 每个 PR 都包含独立勾选项（checklist），可直接在此文档更新进度。
- 执行顺序遵循“低风险先行、跨边界热点优先、UI 大拆分后置”。

```
PR-01（公共底座）
 ├──▶ PR-02（依赖 PR-01 的共享纯函数基础设施）
```

- PR-03、PR-04 与 PR-01 无交叉文件，可并行推进。
- PR-05、PR-06 建议在 PR-01 合入后再启动。

## 2. 每个 PR 通用门禁（必须勾选）

## 3. PR 级别拆分清单

### PR-01：设置类型与共享纯函数收口（低风险）

目标：先消除”类型定义重复 + 基础 helper 分散”，建立后续重构公共底座。

建议范围（示例）：
- `src/components/SettingsPanel.types.ts`
- `src/components/settings/renderSettingsMainSection.types.ts`
- `src/features/backend/mediaLocator.ts`
- `src/features/backend/mappers.ts`
- `src/features/backend/repository/mock/mappers.ts`
- `electron/fileSystemServiceHelpers.ts`
- `src/features/backend/repository/mock/utils.ts`

PR 内检查：
- [ ] 设置面板类型单一来源（Single Source of Truth）已落地（两个 types 文件 >90% 重叠需消除）
- [ ] mapper 公共逻辑抽到共享纯函数，prod/mock 行为对齐
- [ ] `normalizeFeatureFilter`/`matchesFeatureFilter`/`pickSourceGrade` 收敛为单一实现（当前在 `fileSystemServiceHelpers.ts` 和 `mock/utils.ts` 中完全重复）
- [ ] `deriveWorkTitleFromFileName`/`syncPackageNameFromWorkTitle` 收敛为单一实现（当前分散在 3 处）
- [ ] `normalizeMetadataText`/`normalizeTextValue` 及 `normalizeMetadataTags`/`normalizeTags` 对齐合并
- [ ] `mediaLocatorKey` 与 `locatorPathKey` 统一为单一实现
- [ ] helper 收敛后无行为变更（输入输出契约保持一致）
- [ ] 增加/更新对应单元测试

回归检查：
- [ ] 设置页打开、切换、保存后重启值一致
- [ ] 媒体定位（图片/视频/音频）读取结果与改造前一致
- [ ] MockMediaRepository 相关测试全部通过

回滚策略：
- [ ] 保留旧 helper/mappers 兼容导出（一个版本窗口）
- [ ] 若出现线上偏差，优先切回旧映射实现并保留新测试样例

## 4. 本轮不纳入治理范围的重复模式

以下重复模式经评估后**不纳入本轮治理**，记录于此供后续迭代参考。

### 转码服务三件套（预估可消减 800-1000 行）

- `electron/services/file-system-read/managementVideoTranscodeService.ts`（1103 行）
- `electron/services/file-system-read/managementAudioTranscodeService.ts`（951 行）
- `electron/services/file-system-read/managementImageConvertService.ts`（787 行）

三者共享近似的任务状态管理、进度追踪接口、取消处理、文件 I/O 模式。不纳入原因：改动涉及 Electron 主进程核心服务，回归面过大，需在本轮基础 PR 稳定后单独规划。

`src/features/app/build*PanelProps.ts` 系列文件具有相似的参数收集 → 转换 → 输出模式。不纳入原因：每个 builder 的业务逻辑差异较大，强行工厂化收益有限且引入抽象成本，可在 PR-06 拆分大 Hook 后重新评估。

### Mock Repository Handler 类（4 个 Handler 文件）

- `src/features/backend/repository/mock/MediaWriteHandlers.ts`（1052 行）

- `src/features/backend/repository/mock/SystemHandlers.ts`（99 行）

不纳入原因：Mock 实现仅用于测试，不影响生产代码质量，优先级低于生产代码治理。

### IPC Handler Facade 模式

- `electron/facade/FileSystemManagementHandlers.ts`（828 行）
