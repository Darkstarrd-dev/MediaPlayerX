# MediaPlayerX 开发启动清单

Last updated: 2026-02-18

本文档用于保证在任意机器拉取仓库后，都能基于当前基线（`v0.75`）继续稳定开发。

## 0. 使用原则

- 本项目采用“文档单一事实源（SSOT）”原则。
- 需求/架构/交互以 `docs/` 根目录 SSOT 文档为准。
- 历史资料只在 `docs/archive/` 追溯，不作为当前实现依据。
- 代码与文档冲突时，先以当前代码行为确认事实，再同步修订 SSOT 文档。

## 1. 新机器接手（首次）

1. 安装基础环境：Git、Node.js `22.x`、npm。
2. 克隆仓库并进入目录：

```bash
git clone https://github.com/Darkstarrd-dev/MediaPlayerX.git
cd MediaPlayerX
```

3. 对齐主分支：

```bash
git checkout main
git pull
git status
```

4. 按顺序阅读文档：
   - `docs/DOCS_INDEX.md`
   - `docs/requirements-v1.md`
   - `docs/architecture-v1.md`
   - `docs/interaction-v1.md`
   - `docs/backend-integration-guardrails.md`

## 2. 每次开始开发前

1. 同步远端：`git pull`
2. 明确本次改动对应的 SSOT 文档条目。
3. 新建分支（推荐）：

```bash
git checkout -b <type>/<topic>
```

## 3. 开发过程要求

- 改动前先定位对应文档约束，再落代码。
- 保持模块边界与 IPC 合约一致（`contracts -> preload -> ipc -> main/repository`）。
- 行为变化必须同迭代更新文档。
- 提交保持原子化：一个功能点或一个文档变更点。

## 4. 提交前检查

1. 检查变更：

```bash
git status
git diff
```

2. 最小质量门禁（本地）：

```bash
npm run format:check
npm run lint
npm run test
npm run test:coverage
npm run build
```

3. 需要全量对齐 CI 时执行：

```bash
npm run quality:ci
npm run build:electron
```

4. 提交与推送（Conventional Commits）：

```bash
git add <files>
git commit -m "<type>(<scope>): <subject>"
git push -u origin <branch>
```

## 5. 跨机器续开发（非首次）

1. 进入项目目录并拉取最新：`git pull`。
2. 先读 `docs/DOCS_INDEX.md` 与最近修改的 SSOT 文档。
3. 切回工作分支或新建分支继续开发。
4. 开发完成后按第 4 节执行门禁并提交。

## 6. 文档映射规则

- 需求变更：更新 `docs/requirements-v1.md`。
- 架构/模块边界变更：更新 `docs/architecture-v1.md`。
- UI/交互/快捷键变更：更新 `docs/interaction-v1.md`。
- 后端/IPC 接入规则变更：更新 `docs/backend-integration-guardrails.md`。
- i18n/a11y 规则变更：更新 `docs/i18n-aria-guardrails.md`。
- Shader 运行时/迁移规则变更：更新 `docs/music-visualizer-shader-entry.md` 与 `docs/music-visualizer-shader-migration-playbook.md`。
- Theme token/风格规则变更：更新 `docs/theme-system-v2.md` 与 `docs/theme-brainstorm-entry.md`。
- 文档目录结构变更：更新 `docs/DOCS_INDEX.md` 与 `docs/README.md`。

## 7. Windows 开发注意事项

- 不要用“管理员权限”启动 Electron（例如管理员终端运行 `npm run dev:desktop`）。
- 否则会触发 UAC 权限隔离，资源管理器拖拽到窗口会被系统阻断（鼠标禁用图标，drop 无回调）。

## 8. 常用命令

```bash
npm install
npm run dev
npm run dev:desktop
npm run lint
npm run test
npm run build
```
