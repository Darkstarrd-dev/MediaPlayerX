# MediaPlayer 开发启动清单

本文档用于保证在任意机器拉取仓库后，都能基于当前代码与文档继续稳定开发。

## 0. 使用原则

- 本项目采用“文档单一事实源（SSOT）”原则。
- 需求、架构、交互以 `docs/` 下文档为准。
- 代码实现与文档冲突时，先修正代码到文档，或同步评审后更新文档。

## 1. 新机器接手（首次）

1. 安装基础环境：Git、Node.js（LTS）、npm。
2. 克隆仓库：

```bash
git clone https://github.com/Darkstarrd-dev/MediaPlayerX.git
cd MediaPlayerX
```

3. 拉取最新内容并确认分支：

```bash
git checkout main
git pull
git status
```

4. 按顺序阅读核心文档：
   - `docs/requirements-v1.md`
   - `docs/architecture-v1.md`
   - `docs/interaction-v1.md`

## 2. 每次开始开发前

1. 同步远端：

```bash
git pull
```

2. 复核本次开发范围：从文档中确认对应约束（需求/架构/交互）。
3. 建议新建工作分支：

```bash
git checkout -b feat/<topic>
```

## 3. 开发过程要求

- 改功能前先定位对应文档条目。
- 改代码时保持模块边界与 IPC 合约一致。
- 若行为变化，必须同步更新相应文档。
- 提交粒度以“一个明确功能点”或“一个明确文档变更点”为单位。

## 4. 提交前检查

1. 检查变更：

```bash
git status
git diff
```

2. 自检清单：
   - 代码行为与 `requirements-v1.md` 一致。
   - 模块边界与 `architecture-v1.md` 一致。
   - 交互与快捷键行为与 `interaction-v1.md` 一致。
   - 若有行为变化，文档已同步更新。

3. 提交并推送：

```bash
git add .
git commit -m "<type>: <message>"
git push -u origin <branch>
```

## 5. 跨机器续开发（非首次）

1. 进入项目目录并拉取最新：`git pull`。
2. 快速阅读文档更新（优先看最近提交涉及的文档）。
3. 恢复到上次工作分支或新建分支继续开发。
4. 开发完成后按第 4 节流程提交。

## 6. 文档映射规则

- 需求变更：更新 `docs/requirements-v1.md`。
- 架构或模块边界变更：更新 `docs/architecture-v1.md`。
- UI/交互/快捷键变更：更新 `docs/interaction-v1.md`。
- 流程规范变更：更新 `docs/README.md` 与本清单。

## 7. 当前阶段说明

- 当前仓库以产品与架构文档为主。
- 大 I/O 压测在对应实施阶段执行，不在本阶段提前开展。

## 8. 虚拟 UI 阶段常用命令

```bash
npm install
npm run dev
npm run test
npm run lint
```
