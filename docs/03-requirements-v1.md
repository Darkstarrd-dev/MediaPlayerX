# MediaPlayer V1 需求冻结

Last updated: 2026-02-18

## 产品目标

构建一个本地优先的图片/视频综合浏览器，具备高可扩展性、模块化能力与可测试性。

## 技术栈

- 桌面端：Electron + Node.js
- 前端：Vite + React
- 状态管理：Zustand
- 类型与校验：TypeScript + Zod
- 测试：Vitest（单元 + 组件），E2E 由用户按脚本执行
- 元数据与运行时数据库：SQLite
- 检索索引：SQLite + 元数据字段（名称/作品名/社团/作者/标签(tags)/评分）

## 平台与存储范围

- 目标平台：Windows。
- 入库来源：本地磁盘。
- 移动硬盘等可临时浏览，但默认不写入数据库。

## 导入与更新行为

### 支持的输入形式

- 单个图片或视频文件。
- 多个图片或视频文件。
- 单个目录（递归扫描媒体与压缩包）。
- 多个目录（递归扫描）。
- 文件与目录混合选择。

### 支持的输入渠道

- 拖拽到应用窗口。
- 应用内原生文件选择器。
- 粘贴文件路径。

### 压缩包规则

- 核心压缩格式为 `zip`。
- 源文件为 `rar` 或 `7z` 时，走任务管线转换为标准存储 `zip`。
- 持续监控已入库根目录中的新增 `zip`。
- 通过 `size/mtime/hash` 识别 `zip` 变化并触发更新或移除。

## 缩略图规则

- 使用 Sharp 生成。
- 输出格式：WebP。
- 默认质量：`40`。
- 默认缩略图宽度：`512` px（保持原图比例，按宽度约束生成）。
- 默认缩略图间距系数：`1.00`（实际像素按 `round(容器内边距 * 0.8 * 系数)` 运行时计算）。
- 所有缩略图参数均可在设置面板调整。
- 缩略图主网格卡片比例固定 `1:1`，布局需优先贴合可用高度，避免底部裁切与可见滚动。
- 主界面水平分割条拖动过程中不执行自动吸附；仅在拖动结束（`mouseup`）后执行一次吸附。

## 检索能力

- 文本检索：覆盖文件名/图包名及元数据字段。
- 特征检索：名称/作品名/社团/作者/标签(tags)/评分。
- 不提供向量检索控件。
- 视频模式特征检索结果受当前 Sidebar 根目录范围约束。

## 标签与元数据策略

- 标签与检索以已入库元数据为准，不依赖本地 ONNX 自动标签模型。
- 元数据管理保留人工整理与写回流程，不包含自动标签批处理入口。
- AI 模型设置仅保留广告审核视觉模型配置与连通性测试。

## 元数据与人工整理

- 元数据字段预留：
  - `work_title`
  - `work_title_jpn`
  - `series_id`
  - `language`
  - `circle`
  - `circle_jpn`
  - `author`
  - `author_jpn`
  - `tags`
- 图片元数据包含 `grade` 字段，范围 `0..5`，默认 `null`。
- AI 元数据补全先预留数据结构与流程，外部抓取逻辑后续接入。

## 重命名与物理变更策略

- 是否同步物理重命名为可选项。
- 物理同步范围：
  - 图包文件名。
  - 目录名。
- `zip` 内部文件重命名仅作为整理工具。
- `zip` 内部重命名/重排序不影响核心业务语义。

### 批量重命名（新增）

- 统一使用 `renameItems` 契约，支持目标：`sidebar-node`、`image-item`、`archive-entry`。
- 支持模式：`single`、`replace`、`numbering`、`remove-range`、`metadata`。
- 批量执行默认 `fail_fast=true`，任一冲突可直接失败并返回失败明细。
- 必须支持 `preview_only=true` 预览，不落盘，仅返回 `source/target` 结果清单。
- `zip` 内部条目重命名仅允许图片扩展名白名单，执行采用“重打包临时文件 -> 原子替换 -> 失败回滚”。
- metadata 模板默认：`[author.jp(if exist)(author.en(if exist))]/[author(if only one exist)]-[circle just like author ] - [title.jp(if exist)]/[title(if only one exist)]`。

## 移除与冲突策略

- 磁盘文件丢失时，自动从数据库索引移除。
- 用户手动从库移除时，可选择是否同时删除实体文件。
- 出现异常冲突时，弹窗提供：
  - 替换已有内容
  - 更名后保留两份

## UI 相关行为约束

- 图片模式下，Sidebar 展示目录/图包树。
- 视频模式下，Sidebar 展示目录/视频叶子树。
- Main 页脚显示焦点 (focus) 项详细信息：
  - `zip` 图片：`zip 绝对路径 + 序号`
  - 文件大小
  - 分辨率

## 视频模式说明

- 视频模式以视频文件作为 Sidebar 叶子节点。
- 每个视频叶子带开关 (toggle)，用于加入/移出全屏播放列表。
- 播放列表为全库共用，不按当前根 (Current Root) 分割。
- 播放列表可在元数据面板查看、拖拽排序、删除条目。
- 从播放列表删除条目时，同步 Sidebar 对应开关 (toggle) 状态。
- 预览控件包含“保存封面 (Save as cover)”，用于手动保存封面。
- 图片/视频元数据均支持 `series_id`（系列ID）人工录入；当图片与视频的 `series_id` 相同，可在元数据面板进行跨模式跳转。
- 视频模式管理态必须提供任务化视频转码能力：支持批量输入、进度、取消、失败重试与输出文件追踪。
- 视频转码必须提供能力灰度（encoder/muxer）与体积预估（含区间/置信度），且参数变更后可刷新预估。
- 视频转码执行前必须做最小化预检：至少包含磁盘空间不足拦截（按预计输出高水位对比可用空间）与常见 ffmpeg 错误归因。
- 视频转码完成后必须支持一键打开输出目录。

## 音乐可视化说明（新增）

- 音乐模式主区必须提供实时音频可视化，不允许长期占位态。
- 可视化输入由播放器统一提供：时间、帧号、分辨率、音频频谱/波形。
- Shader 资源必须与播放器主体解耦，新增或替换 Shader 不应要求修改 App 主体编排代码。
- 默认渲染后端为 `GPU(WebGL2 Shader)`，并提供 `CPU(Canvas2D fallback)` 作为无硬件加速环境保底。
- 必须支持全屏可视化显示，并保持 `Esc/F` 退出语义一致。
- 设置面板必须提供以下可持久化配置：
  - `实际渲染长边分辨率`
  - `FPS 调试显示开关`
  - `CPU/GPU 渲染后端切换`

## 全屏行为

- 双击或 Enter 进入，全屏由 `F` (`KeyF`) 切换。
- 无常驻控制条；鼠标进入底部 20% 区域时显示浮动 Footer。
- Footer 支持显示模式切换：
  - 双显示
  - 仅视频
  - 仅图片
- 双显示模式下，图片与视频区域可同时展示并可调尺寸。

## 交付策略

- 优先实现架构骨架与关键交互。
- 当前已进入“实链路 + 质量收敛”阶段：真实数据库、真实媒体渲染、真实 IPC 写链路均已接入。
- 交付门禁以 CI 口径为准：`format:check + lint + build + test + test:coverage + audit(high)`。
- 大 I/O 压测保留在 `docs/perf/*` 方案与基准流程中持续执行。

## 已落地扩展能力（当前版本可用）

### 管理模式广告图片审核 (LLM Ad Review)

- 输入来源固定为“管理模式当前勾选对象”（Sidebar 节点或图片勾选），不新增独立选区语义。
- 系统仅输出“疑似广告图片候选列表（待人工确认）”，不得自动执行物理删除。
- 用户在审核列表人工确认后，删除动作复用现有管理模式删除链路（`deleteImageItems`）。
- 审核任务支持策略参数化：`all/head-tail` + 并发度（由设置面板配置并透传到 Main 审核链路）。
- 审核结果包含可解释信息（命中原因）与审计信息（来源分布、LLM 命中率、总体命中率）。
- 支持“已确认删除图片哈希复用”，对重复图片可跳过 LLM 直接标记为疑似广告。
- 详细实现以当前 `contracts/repository/main` 实际代码为准。
